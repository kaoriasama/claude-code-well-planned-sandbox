Claude Code を安全に運用するための非エンジニア向けサンドボックス環境テンプレートです。

プログラミング経験がなくても Claude Code を使い始められることを想定して設計しました。

・速度よりも安全性とユーザーのコントロールを優先する

・プロジェクトごとにサンドボックスを立てる

・計画と実行の分離（Opus 4.5 thinking以上推奨）

・タスクの目的把握からリスク評価、実行中の逸脱対応までをカバーする計画策定フレームワークを持つ

・Agentが柔軟に項目の省略やフェーズの跳躍を判断しながら、ユーザーと対話的に計画を練り上げる

・計画に最大の時間を使う


すべての操作にユーザーの明示的な許可を求める設計になっています。サンドボックスによるファイルシステム分離、全Bashコマンドの許可制実行、Auto-Acceptの不使用を基本方針とし、Claudeが作業ディレクトリ外に影響を与えることを防ぎます。

Boris Cherny 氏の Plan Mode重視の思想（https://x.com/bcherny/status/2007179832300581177?s=20 ）を参考にしつつ、非エンジニア向けの安全で理解負債を小さくする環境を目指し、「計画→承認→実行」の対話的なフローを採用しています。
